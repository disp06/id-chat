# ID Secret Chat

Анонимный веб-чат с end-to-end шифрованием (AES-GCM), частными комнатами и передачей файлов (в разработке). Сообщения хранятся только в памяти и удаляются через час неактивности комнаты.

## 🔐 Особенности

- **E2E шифрование** — сервер не видит содержимого сообщений. Ключи генерируются на клиенте.
- **Приватные комнаты** — идентификатор комнаты + секрет (32 байта). Можно добавить пароль (PBKDF2).
- **Серверeless дизайн** — сервер лишь ретранслирует зашифрованные данные.
- **Авто-очистка** — неактивные комнаты удаляются через 1 час.
- **Docker** — запуск за минуту.
- **Легковесный фронтенд** — чистый JS, Web Crypto API (AES-GCM).

## 🚀 Быстрый запуск (Docker)

```bash
# Скопируйте .env.example в .env и при необходимости измените SECRET_KEY
cp .env.example .env

# Сборка и запуск
docker-compose up -d

# Сервис будет доступен на http://localhost:5537
```

## 🎯 Использование

1. Откройте `http://localhost:5537` в браузере.
2. Нажмите «Generate» для Room ID и Secret (или введите свои).
3. Создайте комнату → появится Room ID и Secret.
4. Поделитесь Room ID и Secret с собеседником (через любой канал).
5. Собеседник вводит те же Room ID и Secret и нажимает «Join Room».
6. Общайтесь! Сообщения шифруются на лету.

**Примечание:** Room ID — короткий идентификатор (6 символов). Secret — 32 случайных байта в base64url. При использовании пароля (опционально) секрет дополнительно защищается PBKDF2.

## 🔧 API

- `GET /` — веб-интерфейс
- `POST /rooms/create` — создание комнаты
  - тело: `{ "password": "" }` (опционально)
  - возвращает: `{ "room_id": "...", "secret": "...", "salt": "..." }` (salt если пароль)
- `POST /rooms/<room_id>/join` — присоединение
  - тело: `{ "secret": "...", "password": "" }`
- `GET /rooms/<room_id>/messages` — последние сообщения (ciphertext)
  - возвращает: `{ "messages": [ { "sender": "...", "ciphertext": "...", "ts": ... } ] }`

## 🛡️ Безопасность

- Шифрование: **AES-GCM** 256-bit (ключ 32 байта).
- Производение ключа: если пароль задан — **PBKDF2** (100k итераций, соль 16 байт).
- Ключ комнаты никогда не отправляется на сервер в открытом виде; клиент использует secret для генерации ключа локально.
- Сервер не хранит ключи; только ciphertext.
- Рекомендуется использовать HTTPS в продакшене (например, через Cloudflare Tunnel или Nginx).

## 🐛 Ограничения

- В этом запуске используется хранение комнат в памяти (при перезапуске сервера комнаты теряются). Для масштабирования нужно перенести `rooms_manager` на Redis.
